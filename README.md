## рҹӣ пёҸ ConfiguraciГіn e InstalaciГіn RГЎpida (Paso a Paso)

Para que puedas correr estos laboratorios de forma local, utilizaremos un entorno nativo: **Windows 10/11** actuarГЎ como el hipervisor fГӯsico (mediante **Hyper-V**) y ejecutarГЎs los comandos de Vagrant desde tu consola de comandos (PowerShell, CMD, o Git Bash).

Sigue esta guГӯa narrativa sencilla para preparar tu computadora en 10 minutos:

### 1. Activar Hyper-V en tu Windows
Hyper-V es la tecnologГӯa nativa de virtualizaciГіn de Windows. Necesitamos activarla:
1. Abre el menГә Inicio de Windows, escribe **PowerShell**, haz clic derecho sobre Г©l y selecciona **Ejecutar como Administrador**.
2. Copia, pega y ejecuta el siguiente comando:
   ```powershell
   Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
   ```
3. Si el sistema te solicita reiniciar para aplicar los cambios, acepta y reinicia tu equipo.

### 2. Instalar Vagrant en Windows
Vagrant es la herramienta que crearГЎ y configurarГЎ la mГЎquina virtual por nosotros:
1. Ve al sitio oficial de descargas de [Vagrant](https://www.vagrantup.com/downloads) y descarga el instalador para Windows (arquitectura AMD64/x86_64).
2. Ejecuta el instalador descargado y sigue el asistente haciendo clic en "Next" hasta finalizar.

### 3. Configurar rsync en el PATH de Windows (Recomendado para evitar credenciales)
Por defecto, al usar Hyper-V, Vagrant intentarГЎ sincronizar las carpetas locales usando **SMB**, lo que te solicitarГЎ ingresar tu usuario y contraseГұa de Windows en cada arranque. Para evitar esta peticiГіn y hacer el proceso 100% silencioso y rГЎpido:
1. AsegГәrate de tener instalado [Git para Windows](https://git-scm.com/download/win).
2. Abre el menГә Inicio de Windows, escribe **variables de entorno** y selecciona **Editar las variables de entorno del sistema**.
3. Haz clic en el botГіn **Variables de entorno...**.
4. En la secciГіn superior ("Variables de usuario"), busca la variable llamada **`Path`**, selecciГіnala y haz clic en **Editar...**.
5. Haz clic en **Nuevo** y aГұade la siguiente ruta (donde Git instala `rsync.exe`):
   ```text
   C:\Program Files\Git\usr\bin
   ```
6. Haz clic en **Aceptar** en todas las ventanas. **Cierra todas tus terminales abiertas** y vuelve a abrir una terminal de PowerShell para que se cargue la nueva configuraciГіn.

## рҹҡҖ CГіmo Iniciar y Usar el Laboratorio

Una vez realizada la instalaciГіn inicial, el uso diario es sumamente rГЎpido:

### Paso A: Clonar el proyecto
Abre la terminal de **PowerShell** en tu Windows 10/11 y clona este repositorio (requiere tener **Git** instalado en Windows) en un directorio nativo (por ejemplo, `C:\proys\`):
```powershell

```

### Paso B: Encender la MГЎquina Virtual
Inicia la mГЎquina AlmaLinux 9 de estudio desde tu consola (PowerShell o Git Bash) con privilegios de Administrador (requerido para Hyper-V):

> [!IMPORTANT]
> **SelecciГіn del Switch Virtual en Hyper-V**:
> Durante el arranque de la mГЎquina virtual, Vagrant te solicitarГЎ elegir un **Virtual Switch** (Switch Virtual). 
> * **RecomendaciГіn**: Selecciona la opciГіn correspondiente a **`Default Switch`**.
> * **Por quГ©**: Este switch interno de Windows viene preconfigurado con asignaciГіn de IP automГЎtica (DHCP) y traducciГіn de red (NAT), lo que asegura que tu mГЎquina virtual obtenga salida a Internet para el aprovisionamiento de dependencias y que Vagrant pueda comunicarse con ella vГӯa SSH.

### Paso C: Entrar a la MГЎquina de Estudio y Ejecutar el Lab
1. Accede a la consola de la mГЎquina virtual vГӯa SSH:

2. Dentro de la mГЎquina (que es un entorno Linux de AlmaLinux 9), navega al directorio de laboratorios `/labs/` y entra en el mГіdulo que desees practicar (por ejemplo, el mГіdulo 01):
   ```bash

   ```
3. Ejecuta la demostraciГіn animada con explicaciones en espaГұol para entender los conceptos:
   ```bash

   ```
4. Lee las instrucciones del reto prГЎctico:
   ```bash
   cat instructions.md
   ```
5. Realiza los cambios necesarios en el subdirectorio `challenge/` para resolver el reto (puedes consultar pistas progresivas con `cat hints.md`).
6. Valida si tu soluciГіn es correcta ejecutando el script evaluador automГЎtico no destructivo:
   ```bash

   ```
7. Si deseas volver a practicar desde cero o limpiar tu entorno, puedes restablecer el reto ejecutando:

