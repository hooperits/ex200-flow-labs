## рҹҡҖ GuГӯa de Inicio RГЎpido (Paso a Paso)

Sigue este flujo cronolГіgico ordenado para configurar tu entorno y ejecutar tu primer laboratorio en menos de 10 minutos.

### Paso 1: Clonar el proyecto
Primero, abre la terminal de **PowerShell** en tu Windows 10/11 y clona este repositorio en un directorio local de tu disco fГӯsico (por ejemplo, `C:\proys\`):
```powershell
# Nota: Requiere tener Git para Windows instalado
git clone https://github.com/hooperits/ex200-flow-labs.git
cd ex200-flow-labs
```

### Paso 2: Activar Hyper-V en Windows
Hyper-V es la tecnologГӯa nativa de virtualizaciГіn de Windows. Para activarla:
1. Con la consola de **PowerShell** aГәn abierta, asegГәrate de ejecutarla con privilegios de **Administrador**.
2. Copia y ejecuta el siguiente comando:
   ```powershell
   Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
   ```
3. Si el sistema te solicita reiniciar para aplicar los cambios, acepta y reinicia tu equipo.

### Paso 3: Instalar Vagrant
Vagrant crearГЎ y gestionarГЎ la mГЎquina virtual de forma automГЎtica:
1. Descarga el instalador oficial de [Vagrant para Windows](https://www.vagrantup.com/downloads) (arquitectura AMD64/x86_64).
2. Ejecuta el archivo descargado y completa el asistente haciendo clic en "Next" hasta finalizar.

### Paso 4: Encender la MГЎquina Virtual
Para evitar ingresar tu cuenta personal de Windows (o si inicias sesiГіn con PIN y no recuerdas tu contraseГұa), crearemos un usuario local auxiliar en Windows exclusivo para la comparticiГіn de archivos del laboratorio:

1. Abre tu terminal de **PowerShell** con privilegios de **Administrador** (requerido por Hyper-V).
2. Crea el usuario local `vagrantlabs` ejecutando los siguientes dos comandos:
   ```powershell
   net user vagrantlabs MiPasswordSeguro123 /add
   net localgroup Administradores vagrantlabs /add
   ```
3. Inicia la mГЎquina virtual de estudio (AlmaLinux 9) ejecutando:
   ```powershell
   vagrant up --provider=hyperv
   ```

> [!IMPORTANT]
> **1. SelecciГіn del Switch Virtual en Hyper-V**:
> Durante el arranque, Vagrant te solicitarГЎ elegir un **Virtual Switch** (Switch Virtual). 
> * **RecomendaciГіn**: Elige el nГәmero de opciГіn correspondiente a **`Default Switch`**.
> * **Por quГ©**: Este switch interno de Windows viene preconfigurado con asignaciГіn de IP automГЎtica (DHCP) y traducciГіn de red (NAT), lo que asegura que tu mГЎquina virtual obtenga salida a Internet para el aprovisionamiento y que Vagrant pueda comunicarse con ella vГӯa SSH.

> [!NOTE]
> **2. Credenciales de Windows (SMB)**:
> Al montar las carpetas, Vagrant te pedirГЎ un usuario y contraseГұa. Utiliza la cuenta de servicio local que acabas de crear:
> * **Username (usuario)**: `vagrantlabs`
> * **Password (contraseГұa)**: `MiPasswordSeguro123`
> *(Nota: Vagrant no almacena tus credenciales en ningГәn sitio; solo las pasa localmente a Windows para autorizar el montaje de la carpeta de red hacia la VM).*


### Paso 5: Acceder a la VM y Ejecutar el Lab
1. ConГ©ctate a la consola de la mГЎquina virtual vГӯa SSH:
   ```powershell
   vagrant ssh
   ```
2. Dentro de la mГЎquina (que es un entorno Linux puro de AlmaLinux 9), navega al directorio del laboratorio y entra en el mГіdulo que desees practicar (por ejemplo, el mГіdulo 01):
   ```bash
   cd /labs/01-essential-tools/
   ```
3. Ejecuta la demostraciГіn animada en espaГұol para ver los comandos en acciГіn:
   ```bash
   ./demo.sh
   ```
4. Lee las instrucciones del reto prГЎctico:
   ```bash
   cat instructions.md
   ```
5. Realiza los cambios necesarios en el subdirectorio `challenge/` para resolver el reto (puedes ver pistas progresivas con `cat hints.md`).
6. Valida si tu soluciГіn es correcta ejecutando el validador automГЎtico no destructivo:
   ```bash
   ./verify.sh
   ```
7. Si deseas volver a practicar desde cero, limpia el entorno ejecutando:
   ```bash
   ./reset.sh
   ```

> [!TIP]
> **SincronizaciГіn de Archivos**:
> Si realizas cambios en las instrucciones o scripts de la carpeta `./labs/` en el host (Windows) con tu editor de cГіdigo (como VS Code), puedes sincronizarlos con la mГЎquina virtual en cualquier momento ejecutando desde PowerShell en Windows:
> ```powershell
> vagrant provision
> ```
